# Plano de Auditoria e Hardening de Segurança — oraculoai-site

## Resumo Executivo

Auditoria completa do projeto oraculoai-site (Next.js 16, Vercel, Supabase, HuggingFace AI). O projeto já possui uma postura de segurança sólida (CSP, CSRF, rate limiting, validação Zod, RLS no Supabase), mas foram identificados **2 problemas de alta gravidade**, **3 de média** e **2 de baixa** que precisam ser corrigidos.

**Nenhuma mudança de schema Supabase necessária. Zero breaking changes.**

---

## Achados por Prioridade

### P0 — Rate Limit com Configuração Errada (ALTO)
- **Arquivo**: `src/lib/rate-limit.ts:56-59`
- **Problema**: O closure do Supabase RPC usa valores hardcoded (`p_limit: 10`, `p_window_ms: 10_000`) em vez de usar `opts.limit` e `opts.windowMs` do chamador. O endpoint de chat pretende 20 msgs/hora mas na prática aplica 10 msgs/10s.
- **Fix**: Alterar o tipo e closure de `supabaseRpc` para aceitar `opts: RatelimitOpts` e repassá-los ao RPC.

### P1 — Middleware CSP Não Ativo (ALTO)
- **Arquivo**: `src/proxy.ts`
- **Problema**: O arquivo exporta `proxy()` em vez de `middleware()`. Next.js 16 App Router requer export `middleware` em `src/middleware.ts`. O CSP nonce-based para API routes é código morto.
- **Fix**: Renomear `src/proxy.ts` → `src/middleware.ts`, trocar `proxy` por `middleware`.

### P2 — CSRF Bypass sem Origin/Referer (MÉDIO)
- **Arquivo**: `src/lib/csrf.ts:55-56`
- **Problema**: Requisições sem Origin e sem Referer retornam `true` (confiam). Um atacante pode forjar POST requests sem esses headers.
- **Fix**: Rejeitar POST sem Origin/Referer. Permitir GET sem Origin (navegadores não enviam Origin em GETs).

### P3 — Rate Limit In-Memory Ineficaz no Vercel (MÉDIO)
- **Arquivo**: `src/lib/rate-limit.ts:19-31`
- **Problema**: O fallback in-memory com `Map` reseta a cada cold start do serverless. É um no-op no Vercel.
- **Fix**: Transformar em no-op explícito com warning, em vez de simular proteção inexistente.

### P4 — Dados de Lead Logados (BAIXO)
- **Arquivo**: `src/app/api/leads/route.ts:86`
- **Problema**: PII (nome, email, empresa, mensagem) logado quando Supabase não configurado.
- **Fix**: Mascarar dados no log.

### P5 — Sem Headers CORS Explícitos (MÉDIO)
- **Arquivo**: `src/middleware.ts` (após rename)
- **Problema**: Nenhuma configuração CORS. Mesmo-origin por padrão, mas headers explícitos seriam defense-in-depth.
- **Fix**: Adicionar CORS headers no middleware para rotas da API.

### P6 — Sem .npmrc (BAIXO)
- **Arquivo**: `.npmrc` (novo)
- **Fix**: Criar com `audit=true`, `fund=false`.

### P7 — Vulnerabilidade PostCSS (INFO)
- Build-time apenas, baixo risco. Aguardar fix upstream.

## Plano de Implementação

### Fase 1: Corrigir Rate Limit (P0)

**Arquivo**: `src/lib/rate-limit.ts`

1. **Linha 35** — Mudar tipo do closure:
```typescript
// DE:
let supabaseRpc: ((identifier: string) => Promise<RatelimitResult>) | null = null;
// PARA:
let supabaseRpc: ((identifier: string, opts: RatelimitOpts) => Promise<RatelimitResult>) | null = null;
```

2. **Linha 48** — Atualizar closure para aceitar e usar `opts`:
```typescript
return async (identifier: string, opts: RatelimitOpts): Promise<RatelimitResult> => {
  // ... getSupabase check unchanged ...
  const { data, error } = await (supabase.rpc as any)("check_rate_limit", {
    p_identifier: identifier,
    p_limit: opts.limit,        // era: 10
    p_window_ms: opts.windowMs, // era: 10_000
  });
  // ... error fallback usa opts também ...
};
```

3. **Linha 104-105** — Repassar `opts`:
```typescript
if (rl) {
  return rl(identifier, opts);  // era: rl(identifier)
}
```

**Verificação**: Deploy, testar chat (20 msgs/hora), leads (5/min), hibp (60/min).

### Fase 4: Rate Limit Fallback Explícito (P3)

**Arquivo**: `src/lib/rate-limit.ts`

Substituir `inMemoryCheck` (linhas 21-31) por no-op:
```typescript
function inMemoryCheck(_key: string, opts: RatelimitOpts): RatelimitResult {
  // In-memory é ineficaz no Vercel (cold start reseta o Map).
  // Retorna sucesso para não bloquear tráfego legítimo com falsa sensação de segurança.
  return { success: true, remaining: opts.limit };
}
```

E adicionar warning em `tryInitSupabase` quando Supabase não configurado.
